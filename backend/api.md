# API для работы с базой данных онлайн-платформы обучения

## Общая информация

Этот документ описывает API для работы с базой данных онлайн-платформы обучения. API предоставляет возможность управления пользователями, лекциями, тестами и отслеживания прогресса пользователей.

## Базовые URL

- Для локальной разработки: `http://localhost:5000/api`

## Аутентификация

Большинство запросов требуют аутентификации через JWT-токен. Токен должен передаваться в заголовке `Authorization` в формате `Bearer <token>`.

## Эндпоинты

### Пользователи

#### Регистрация пользователя
- **URL**: `/api/register`
- **Метод**: `POST`
- **Описание**: Регистрация нового пользователя
- **Тело запроса**:
  ```json
  {
    "username": "string",
    "email": "string",
    "password": "string"
  }
  ```
- **Ответы**:
  - `201 Created`: Пользователь успешно создан
  - `400 Bad Request`: Некорректные данные
  - `409 Conflict`: Пользователь с таким именем или email уже существует

#### Вход пользователя
- **URL**: `/api/login`
- **Метод**: `POST`
- **Описание**: Аутентификация пользователя
- **Тело запроса**:
  ```json
  {
    "username": "string",
    "password": "string"
  }
  ```
- **Ответы**:
  - `200 OK`: Успешная аутентификация
    ```json
    {
      "token": "string"
    }
    ```
  - `400 Bad Request`: Некорректные данные
  - `401 Unauthorized`: Неверные учетные данные

### Лекции

#### Получение списка лекций
- **URL**: `/api/lectures`
- **Метод**: `GET`
- **Описание**: Получение списка всех доступных лекций
- **Заголовки**: Не требует аутентификации
- **Ответы**:
  - `200 OK`: Успешное получение списка
    ```json
    [
      {
        "id": 1,
        "title": "string",
        "slug": "string",
        "file_path": "string",
        "description": "string",
        "created_at": "timestamp"
      }
    ]
    ```

#### Отметка о просмотре лекции
- **URL**: `/api/lecture-progress`
- **Метод**: `POST`
- **Описание**: Отметка о просмотре лекции пользователем
- **Заголовки**: Требует аутентификации
- **Тело запроса**:
  ```json
  {
    "lecture_id": 1
  }
  ```
- **Ответы**:
  - `200 OK`: Прогресс успешно записан
  - `400 Bad Request`: Некорректные данные
  - `401 Unauthorized`: Требуется аутентификация

### Тесты

#### Получение списка тестов
- **URL**: `/api/tests`
- **Метод**: `GET`
- **Описание**: Получение списка всех доступных тестов
- **Заголовки**: Требует аутентификации
- **Ответы**:
  - `200 OK`: Успешное получение списка
    ```json
    [
      {
        "id": 1,
        "title": "string",
        "description": "string",
        "pass_threshold": 80,
        "created_at": "timestamp"
      }
    ]
    ```

#### Получение вопросов теста
- **URL**: `/api/tests/{test_id}/questions`
- **Метод**: `GET`
- **Описание**: Получение вопросов для указанного теста
- **Заголовки**: Требует аутентификации
- **Ответы**:
  - `200 OK`: Успешное получение вопросов
    ```json
    [
      {
        "id": 1,
        "question_text": "string",
        "options": [
          {
            "id": 1,
            "option_text": "string"
          }
        ]
      }
    ]
    ```
  - `404 Not Found`: Тест не найден

#### Сохранение результатов теста
- **URL**: `/api/test-results`
- **Метод**: `POST`
- **Описание**: Сохранение результатов прохождения теста
- **Заголовки**: Требует аутентификации
- **Тело запроса**:
  ```json
  {
    "test_id": 1,
    "answers": [
      {
        "question_id": 1,
        "selected_option_id": 1
      }
    ]
  }
  ```
- **Ответы**:
  - `200 OK`: Результаты успешно сохранены
    ```json
    {
      "score": 8,
      "total_questions": 10,
      "percentage": 80,
      "passed": true
    }
    ```
  - `400 Bad Request`: Некорректные данные

#### Получение результатов теста
- **URL**: `/api/test-results/{test_id}`
- **Метод**: `GET`
- **Описание**: Получение результатов прохождения указанного теста текущим пользователем
- **Заголовки**: Требует аутентификации
- **Ответы**:
  - `200 OK`: Успешное получение результатов
    ```json
    {
      "score": 8,
      "total_questions": 10,
      "percentage": 80,
      "passed": true,
      "completed_at": "timestamp"
    }
    ```
  - `404 Not Found`: Результаты не найдены

## Структура базы данных

### Таблица `users`
Хранит информацию о зарегистрированных пользователях.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор пользователя |
| username | VARCHAR(50) | Имя пользователя (уникальное) |
| email | VARCHAR(100) | Email пользователя (уникальный) |
| password_hash | VARCHAR(255) | Хэш пароля пользователя |
| role | ENUM('user', 'admin') | Роль пользователя |
| created_at | TIMESTAMP | Дата и время создания записи |
| updated_at | TIMESTAMP | Дата и время последнего обновления записи |

### Таблица `lectures`
Хранит информацию о доступных лекциях.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор лекции |
| title | VARCHAR(100) | Название лекции |
| slug | VARCHAR(100) | Уникальный идентификатор для URL |
| file_path | VARCHAR(255) | Путь к файлу лекции |
| description | TEXT | Описание лекции |
| created_at | TIMESTAMP | Дата и время создания записи |

### Таблица `lecture_progress`
Отслеживает прогресс пользователей по лекциям.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор записи |
| user_id | INT (FK) | Идентификатор пользователя |
| lecture_id | INT (FK) | Идентификатор лекции |
| completed | BOOLEAN | Статус завершения лекции |
| viewed_at | TIMESTAMP | Дата и время последнего просмотра |
| updated_at | TIMESTAMP | Дата и время последнего обновления записи |

### Таблица `tests`
Хранит информацию о доступных тестах.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор теста |
| title | VARCHAR(255) | Название теста |
| description | TEXT | Описание теста |
| pass_threshold | INT | Порог прохождения в процентах |
| created_at | TIMESTAMP | Дата и время создания записи |
| updated_at | TIMESTAMP | Дата и время последнего обновления записи |

### Таблица `test_questions`
Хранит вопросы для тестов.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор вопроса |
| test_id | INT (FK) | Идентификатор теста |
| question_text | TEXT | Текст вопроса |
| correct_answer_index | INT | Индекс правильного ответа (0-based) |
| created_at | TIMESTAMP | Дата и время создания записи |
| updated_at | TIMESTAMP | Дата и время последнего обновления записи |

### Таблица `question_options`
Хранит варианты ответов для вопросов тестов.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор варианта ответа |
| question_id | INT (FK) | Идентификатор вопроса |
| option_text | TEXT | Текст варианта ответа |
| option_index | INT | Порядковый номер варианта ответа |
| created_at | TIMESTAMP | Дата и время создания записи |
| updated_at | TIMESTAMP | Дата и время последнего обновления записи |

### Таблица `test_results`
Хранит результаты прохождения тестов пользователями.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор результата |
| user_id | INT (FK) | Идентификатор пользователя |
| test_id | INT (FK) | Идентификатор теста |
| score | INT | Количество правильных ответов |
| total_questions | INT | Общее количество вопросов |
| percentage | DECIMAL(5,2) | Процент правильных ответов |
| passed | BOOLEAN | Результат прохождения (успешно/неуспешно) |
| completed_at | TIMESTAMP | Дата и время завершения теста |

### Таблица `user_answers`
Хранит ответы пользователей на вопросы тестов.

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT (PK) | Уникальный идентификатор ответа |
| user_id | INT (FK) | Идентификатор пользователя |
| test_id | INT (FK) | Идентификатор теста |
| question_id | INT (FK) | Идентификатор вопроса |
| selected_option_id | INT (FK) | Идентификатор выбранного варианта ответа |
| is_correct | BOOLEAN | Правильность ответа |
| answered_at | TIMESTAMP | Дата и время ответа |

## Индексы

Для оптимизации запросов в базе данных созданы следующие индексы:

- `idx_lecture_progress_user_id` на таблице `lecture_progress` по полю `user_id`
- `idx_lecture_progress_lecture_id` на таблице `lecture_progress` по полю `lecture_id`
- `idx_test_results_user_id` на таблице `test_results` по полю `user_id`
- `idx_test_results_test_id` на таблице `test_results` по полю `test_id`
- `idx_user_answers_user_id` на таблице `user_answers` по полю `user_id`
- `idx_user_answers_test_id` на таблице `user_answers` по полю `test_id`
- `idx_user_answers_question_id` на таблице `user_answers` по полю `question_id`

## Заключение

Этот API предоставляет полный набор функций для управления онлайн-платформой обучения, включая регистрацию пользователей, управление лекциями и тестами, а также отслеживание прогресса пользователей.
