# Архитектура онлайн-платформы обучения

## Общее описание

Онлайн-платформа обучения предназначена для предоставления пользователям доступа к обучающим материалам в виде лекций и тестов. Платформа позволяет пользователям регистрироваться, проходить лекции, сдавать тесты и отслеживать свой прогресс.

## Архитектурные компоненты

### 1. Клиентская часть (Frontend)

Клиентская часть реализована с использованием фреймворка Svelte и предоставляет пользовательский интерфейс для:

- Регистрации и аутентификации пользователей
- Просмотра списка лекций
- Просмотра обучающих видео
- Прохождения тестов
- Отслеживания прогресса обучения

#### Основные компоненты:

- **App.svelte**: Главный компонент приложения
- **components/LectureManagement.svelte**: Управление лекциями
- **components/TestManagement.svelte**: Управление тестами
- **routes/**: Маршруты приложения
  - `index.svelte`: Главная страница
  - `lectures.svelte`: Список лекций
  - `testing.svelte`: Список тестов
  - `test-template/test-template.svelte`: Шаблон прохождения теста
  - `lec-*.svelte`: Страницы отдельных лекций

### 2. Серверная часть (Backend)

Серверная часть реализована на Node.js с использованием Express и предоставляет REST API для работы с данными.

#### Основные компоненты:

- **app.js**: Главный файл приложения, инициализация сервера
- **db.js**: Подключение к базе данных MySQL
- **initDb.js**: Инициализация базы данных
- **routes/**: Маршруты API
  - `auth.js`: Маршруты аутентификации
- **middleware/**: Промежуточное ПО
  - JWT-аутентификация
  - Обработка ошибок

### 3. База данных (MySQL)

База данных хранит всю информацию о пользователях, лекциях, тестах и прогрессе пользователей.

#### Основные таблицы:

- **users**: Пользователи системы
- **lectures**: Лекции
- **lecture_progress**: Прогресс по лекциям
- **tests**: Тесты
- **test_questions**: Вопросы тестов
- **question_options**: Варианты ответов
- **test_results**: Результаты тестов
- **user_answers**: Ответы пользователей

### 4. Статические файлы

Платформа предоставляет доступ к статическим файлам:

- Видео лекций в папке `videos/`
- Другие статические ресурсы

## Поток данных

### Регистрация пользователя

1. Пользователь отправляет запрос на `/api/register` с данными регистрации
2. Сервер проверяет уникальность имени пользователя и email
3. Сервер хэширует пароль и сохраняет пользователя в базе данных
4. Возвращается подтверждение успешной регистрации

### Аутентификация пользователя

1. Пользователь отправляет запрос на `/api/login` с учетными данными
2. Сервер проверяет учетные данные
3. При успешной аутентификации сервер генерирует JWT-токен
4. Токен возвращается пользователю для последующих запросов

### Просмотр лекций

1. Пользователь запрашивает список лекций через `/api/lectures`
2. Сервер возвращает список лекций из базы данных
3. Пользователь выбирает лекцию для просмотра
4. При завершении просмотра отправляется запрос на `/api/lecture-progress` для отметки о просмотре

### Прохождение тестов

1. Пользователь запрашивает список тестов через `/api/tests`
2. Сервер возвращает список тестов из базы данных
3. Пользователь выбирает тест для прохождения
4. Запрашиваются вопросы теста через `/api/tests/{test_id}/questions`
5. Пользователь отвечает на вопросы
6. Результаты отправляются через `/api/test-results`
7. Результаты сохраняются в базе данных

## Безопасность

### Аутентификация

- Используется JWT (JSON Web Tokens) для аутентификации пользователей
- Пароли хэшируются с использованием bcrypt
- Токены имеют ограниченное время жизни (7 дней)

### Защита API

- Все защищенные эндпоинты проверяют наличие и валидность JWT-токена
- Используется CORS для ограничения доступа с других доменов
- Валидация входных данных на стороне сервера

## Масштабируемость

### Горизонтальное масштабирование

- Возможность запуска нескольких экземпляров сервера
- Использование балансировщика нагрузки
- База данных может быть вынесена на отдельный сервер

### Вертикальное масштабирование

- Увеличение ресурсов сервера (CPU, RAM)
- Оптимизация запросов к базе данных
- Использование кэширования

## Развертывание

### Локальная разработка

- Node.js и npm для запуска сервера
- MySQL для базы данных
- Файлы конфигурации в `.env`

### Продакшн

- Использование PM2 для управления процессами Node.js
- Настройка reverse proxy (nginx)
- Резервное копирование базы данных
- Мониторинг производительности

## Технический стек

### Frontend

- Svelte - фреймворк для создания пользовательского интерфейса
- HTML5/CSS3 - разметка и стилизация
- JavaScript (ES6+) - язык программирования

### Backend

- Node.js - среда выполнения JavaScript
- Express.js - веб-фреймворк
- MySQL2 - драйвер для работы с MySQL
- bcrypt - библиотека для хэширования паролей
- jsonwebtoken - библиотека для работы с JWT
- dotenv - управление переменными окружения

### База данных

- MySQL - реляционная система управления базами данных

### Инструменты разработки

- nodemon - автоматическая перезагрузка сервера при изменениях
- npm - менеджер пакетов

## Заключение

Архитектура платформы обучения спроектирована с учетом принципов масштабируемости, безопасности и удобства использования. Разделение на клиентскую и серверную части позволяет эффективно развивать каждую из них независимо, а использование современных технологий обеспечивает высокую производительность и надежность системы.
